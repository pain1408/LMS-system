🏛️ CUET Library Management System

A simple, static HTML/CSS/JS library management system that can be deployed anywhere!

📁 Project Structure

cuet-library-static/
├── index.html              # Landing page
├── login.html              # Login page
├── register.html           # Registration page
├── dashboard.html          # Student dashboard
├── books.html              # Books catalog
├── admin.html              # Admin dashboard
├── styles/
│   └── main.css            # All styles
├── scripts/
│   ├── main.js             # Common functionality
│   ├── auth.js             # Authentication
│   ├── dashboard.js        # Dashboard logic
│   ├── books.js            # Books functionality
│   └── admin.js            # Admin functionality
├── images/
│   └── cuet-logo.png       # Logo (add your own)
└── README.html             # This file

✨ Features

📚 Book Management: Browse, search, and pre-book books
👤 User Authentication: Student and admin login
📊 Dashboard: Personal stats and activity tracking
💰 Fine Management: View and pay fines online
📝 Notes Sharing: Upload and download study notes
⚡ Real-time Search: Instant book filtering
📱 Responsive Design: Works on all devices

🚀 Deployment Options

1. GitHub Pages (Free)

  1. Create a new repository on GitHub
  2. Upload all files to the repository
  3. Go to Settings → Pages
  4. Select "Deploy from a branch" → main
  5. Your site will be live at https://username.github.io/repository-name

2. Netlify (Free)

  1. Go to netlify.com
  2. Drag and drop your project folder
  3. Get instant deployment with custom domain

3. Vercel (Free)

  1. Go to vercel.com
  2. Import your GitHub repository
  3. Automatic deployment on every push

4. Any Web Server

Simply upload all files to any web hosting service. No build process required!

🛠️ Technical Details

  • Frontend: Pure HTML5, CSS3, Vanilla JavaScript
  • Icons: Lucide (CDN)
  • Storage: localStorage for demo data
  • Responsive: CSS Grid & Flexbox
  • No Dependencies: No npm, no build process

🔄 Adding Real Backend

To connect to a real database:

  1. Replace localStorage calls with API calls
  2. Update authentication in scripts/auth.js
  3. Modify data loading functions
  4. Add proper error handling

📞 Support

This is a static demo. For a production system, you would need:

  • Backend API (Node.js, PHP, Python, etc.)
  • Database (MySQL, PostgreSQL, MongoDB)
  • User authentication system
  • File upload handling
  • Payment gateway integration
